Source
Meeting §267–286
Status note
In active discussion with Sean / Austin

Determine whether Proxmox + Apache Spark configurations can be driven by hostnames (resolved via DNS) instead of literal IPs. The cleanest way to give Ksolves cluster access without exposing them to corp network is to re-IP the cluster into the KSolves business-partner VLAN, do install work there, then re-IP back to corp at production cutover. This only works if every config tolerates a hostname → IP swap.

User actions
Audit Proxmox cluster config files for hardcoded IPs vs hostnames
Audit Spark / YARN configs (yarn-site.xml, spark-defaults.conf) for hardcoded IPs
Audit Ceph monmap and OSD config for hardcoded IPs
Document where IPs are unavoidable (i.e. would need config edits + restart on re-IP)
Decide go / no-go on the VLAN-move approach based on findings
