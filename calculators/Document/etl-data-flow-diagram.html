Stage-by-Stage: Data Volumes & Constraints

#StageData VolumeKey Constraint / Formula
1 SRCExternal Data Source
Raw CSV from upstream systems
 2–5 TB/day Sets DAILY_TB — all downstream sizing flows from this.
2 CEPHCeph RGW — CSV Landing
s3a://bucket/csv/ · CSV cleared after job begins
 1 day = DAILY_TB CEPH_EFFECTIVE = CEPH_USABLE − VM_OS_TB − PARQ_PENDING_TB
CEPH_DAYS = CEPH_EFFECTIVE / DAILY_TB
3 SPARKMap Tasks — Transform & Shuffle Write
Filter · join · transform · writes sorted shuffle output to NVMe scratch
 Map output = DAILY_TB × shuffle_amp
Persists until all reducers complete		 Snappy compression: 40–60% reduction on disk. KryoSerializer reduces record size. lz4.blockSize=524288 gives −20% additional file size.
4 SCRATCHNVMe Scratch — Map Output + Reduce Spill
Both phases coexist on disk at peak — this is the ×2 factor
 Peak = DAILY_TB × shuffle_amp × 2 Config I ceiling: 4.61× at 5 TB/day. shuffle.memoryFraction=0.30 reduces reduce-side spill. Planning uses raw volume (conservative — compression is a buffer, not a guarantee).
5 SPARKReduce Tasks — Aggregate & Write Parquet
Fetch shuffle blocks · merge · build Parquet output files
 Output ≈ DAILY_TB ÷ 5
~0.4–1 TB/day Parquet		 Target ~150 MB files for Snowflake COPY INTO. df.repartition(OUTPUT_PARTS). Files <64 MB or >300 MB trigger warnings.
6 CEPHCeph RGW — Parquet Outputs
N=2 sets accumulate until prior transfer confirmed
 PARQ_PENDING_TB = N × (DAILY_TB ÷ PARQ_COMP) N_PARQUET slider (1–5, default 2). Each set deducted from CEPH_EFFECTIVE. Transfer pipeline must run ≥1×/day.
7 DESTSnowflake — COPY INTO
Cloud-side ingest · Parquet deleted only on confirmed delivery
 ~0.4–1 TB/day Parquet Stalled transfer pipeline increases N_PARQUET pressure on Ceph headroom until CEPH_DAYS drops below the 3-day warning threshold.
