Fixed Overhead Reservations (per node)

Before any Spark worker VM can be sized, a fixed block of cores and RAM is reserved on every node by the hypervisor, Ceph services, and YARN infrastructure. These resources are not available to worker VMs. The OSD row scales with the NVMe → Ceph OSDs slider. The YARN services VM (ResourceManager, History Server, Hive Metastore, PostgreSQL, Airflow) runs on one node and replaces one worker VM slot. Inside each worker VM, the YARN NodeManager daemon reserves additional cores and RAM before executor sizing. The Effective max / VM row is the true ceiling for the vCPU and RAM sliders.
