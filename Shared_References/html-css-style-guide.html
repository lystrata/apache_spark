HTML Skeleton

All files are single-file HTML with no external dependencies. No build step.

<!DOCTYPE html> <html lang="en" data-theme="dark"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Page Title</title> <style> /* All CSS inline */ :root { ... } /* CSS variables — dark defaults */ [data-theme="light"] { ... } /* CSS variables — light overrides */ /* component styles */ </style> </head> <body id="top"> <div class="page-header"> <div class="page-header-inner"> <div class="header-text"> <h1>Title</h1> <h2>Subtitle</h2> <p>Description</p> </div> <button class="btn-theme" id="btn_theme" onclick="toggleTheme()">&#9728; Light</button> </div> </div> <div class="page-wrap"> <!-- sections --> </div> <script> /* All JS inline — recalc(), theme, state, etc. */ </script> </body> </html>
Serving Rule

Always serve via HTTP — never open as file://. localStorage scoping differs between origins.

python3 -m http.server 8000 # then open http://localhost:8000/calculators/Document/filename.html

Port 8080 is also used in some sessions. Either works — pick one per session and stay consistent.

Staging Rule

All new and refactored files go to Ready_For_Review/ first. After user approval, copy to Document/ and delete from staging. Update calculators/Document/index.html when promoting any new file to Document/.

Elements That Need Manual Light Overrides

Some elements use hardcoded RGBA or fixed colors that don't pick up variable changes automatically. Always add explicit [data-theme="light"] rules for:

  • .warn-item.error/warning/ok/info — RGBA backgrounds
  • Bar segments with !important background overrides
  • .spark-conf-pre token colors (.cc, .ck, .cv)
  • Badge backgrounds (.badge-ok, .badge-warn, .badge-crit)
  • Body grid pattern (background-image)
  • Page header gradient
  • input[type=range] accent color
  • select and input[type=number] backgrounds

recalc() Execution Order Critical load order for slider reads and derived math
This order is load-bearing. Pre-scale block must run before any reads.
function recalc() { /* 1. Pre-scale block (must be first) Recompute dependent slider max from parent slider, then rescale value to maintain percentage position. Example: ram_per_vm max depends on ram_node current value. */ var ramNode = parseInt(el('ram_node').value); var ramPerVmEl = el('ram_per_vm'); var pct = ramPerVmEl.value / ramPerVmEl.max; ramPerVmEl.max = ramNode - INFRA_RAM; ramPerVmEl.value = Math.round(pct * ramPerVmEl.max); /* 2. Slider display updates */ setVal('ram_node', ramNode + ' GB'); setVal('ram_per_vm', parseInt(el('ram_per_vm').value) + ' GB'); /* 3. Read all inputs into constants */ var VMS_NODE = parseInt(el('vms_node').value); var VCPU_VM = parseInt(el('vcpu_vm').value); /* ... */ /* 4. Infrastructure reservation math */ /* 5. Executor math */ /* 6. UI updates (bars, metrics, tables, warnings) */ renderWarnings(...); }

Theme Toggle applyTheme · toggleTheme · localStorage persistence
Each calculator uses a unique localStorage key to avoid cross-page theme conflicts. Use spark_theme_<context>.
/* HTML — place in page-header-inner, sibling of .header-text */ <button class="btn-theme" id="btn_theme" onclick="toggleTheme()">&#9728; Light</button> /* JS — place at end of script block */ function applyTheme(theme) { document.documentElement.setAttribute('data-theme', theme); localStorage.setItem('spark_theme_<context>', theme); document.getElementById('btn_theme').textContent = theme === 'dark' ? '\u2600 Light' : '\u263e Dark'; } function toggleTheme() { var current = document.documentElement.getAttribute('data-theme') || 'dark'; applyTheme(current === 'dark' ? 'light' : 'dark'); } /* Run immediately — before DOMContentLoaded — to avoid flash */ applyTheme(localStorage.getItem('spark_theme_<context>') || 'dark');

localStorage keys in use: spark_theme (dev calc), spark_theme_prod (prod calc), spark_theme_phase1 (phase1 model).

State Persistence (localStorage) saveState · loadState · key conventions
/* localStorage key convention */ spark_calc_dev /* dev calculator session continuity */ spark_calc_prod /* prod calculator session continuity */ spark_calc_dev_phase1 /* phase1 model session continuity */ spark_calc_dev_scenarios /* dev named scenarios */ spark_calc_prod_scenarios /* prod named scenarios */ /* State save — only on explicit Save button or slider change */ function saveState() { try { var sliders = {}; document.querySelectorAll('input[type=range]').forEach(function(s) { sliders[s.id] = s.value; }); localStorage.setItem(LS_KEY, JSON.stringify({sliders: sliders})); } catch(e) {} } /* State restore — ram_node MUST be restored before other sliders because the pre-scale block in recalc() reads it to set ram_per_vm max */ function loadState() { try { var raw = localStorage.getItem(LS_KEY); if (!raw) return; var state = JSON.parse(raw); if (state.sliders) { /* restore ram_node first */ if (state.sliders.ram_node !== undefined) el('ram_node').value = state.sliders.ram_node; /* then all others */ document.querySelectorAll('input[type=range]').forEach(function(s) { if (s.id !== 'ram_node' && state.sliders[s.id] !== undefined) s.value = state.sliders[s.id]; }); } } catch(e) {} }

Cross-Section Jump Links jumpTo() — open target section and scroll to anchor
Never use bare href="#anchor" for cross-section links — target section will be closed and anchor invisible.
<a href="#anchor-id" onclick="jumpTo('anchor-id', 'sec_target')">link text</a> /* anchor target — add id to the element being linked to */ <h3 id="anchor-id">Target Heading</h3> /* JS — one copy per file */ var _jumpedTo = null; function jumpTo(hash, sectionId) { event.preventDefault(); _jumpedTo = {hash: hash, sectionId: sectionId}; history.pushState(null, '', '#' + hash); var d = document.getElementById(sectionId); if (d) d.open = true; setTimeout(function() { var t = document.getElementById(hash); if (t) t.scrollIntoView({behavior:'smooth'}); }, 50); } window.addEventListener('popstate', function() { if (_jumpedTo && location.hash !== '#' + _jumpedTo.hash) { var sec = document.getElementById(_jumpedTo.sectionId); if (sec) sec.open = false; _jumpedTo = null; } });

Security Helper — _escHtml() Required for any user-supplied string injected into innerHTML
Apply to scenario names, file names, or any user input before innerHTML injection.
function _escHtml(s) { return String(s) .replace(/&/g, '&amp;') .replace(/</g, '&lt;') .replace(/>/g, '&gt;') .replace(/"/g, '&quot;'); } /* short alias used inside renderSparkConf / similar */ function esc(s) { return _escHtml(s); }

DOM Helpers el · setTxt · setVal · setHTML
function el(id) { return document.getElementById(id); } function setTxt(id, v) { var e = el(id); if (e) e.textContent = v; } function setVal(id, v) { var e = el(id); if (e) e.textContent = v; } /* alias */ function setHTML(id, v) { var e = el(id); if (e) e.innerHTML = v; }

setHTML is for trusted content only. User-supplied strings must go through _escHtml() before being passed to setHTML.

Key Sizing Formulas
/* Executor memory sizing */ SLOT_GB = (RAM_VM - 2) / EXEC_PER_VM /* 2 GB = NodeManager headroom */ MAX_EXEC_MEM = (SLOT_GB - OFFHEAP) / (1 + OVH_PCT) EXEC_MEM = floor(MAX_EXEC_MEM * 0.95) /* 5% safety margin */ OOM_MARGIN = SLOT_GB - (EXEC_MEM * (1 + OVH_PCT) + OFFHEAP) /* Executor count */ EXEC_PER_VM = floor((VCPU_VM - 2) / EXEC_CORES) /* 2 vCPUs: OS + NodeManager */ TOTAL_EXEC = max(1, EXEC_PER_VM * TOTAL_VMS - 1) /* -1 = YARN cluster-mode driver */ /* Storage */ CEPH_USABLE = OSD_NODE * NVME_TB * NODES / CEPH_REP /* ÷3 for 3× replication */ SCRATCH_PER_VM = (NVME_COUNT - OSD_NODE) / VMS_NODE * NVME_TB

F Conventions & Rules · revisions section · git · sanitization · index update ↑ Top
Revisions Section — When to Update

Any HTML file with a sec_revisions or revisions section must have that section updated in the same edit whenever a formula, calculation, or content change is made.

Do not update for navigation, layout, or styling changes alone.

Files with Revisions sections: development_spark_calculator.html · production_spark_calculator.html · dev_cluster_math_reference.html · dev-cluster-storage-reference.html

Corporate Identifier Sanitization

Before committing any file, replace corporate identifiers (FQDN, company name) with the literal placeholder fqdn. Patterns stored in ~/.config/spark-hooks/patterns. A pre-commit hook enforces this and will block commits with matches.

Index Update Rule

When any new HTML file is promoted to calculators/Document/, add a corresponding entry to calculators/Document/index.html in the appropriate section (Calculators, Guides, Reference, or Research & Correspondence) as part of the same commit.

Theme Toggle Rule

All new calculator and reference HTML files must include the light/dark theme toggle. Use data-theme="dark" on <html> as default. Use the warm amber palette for light mode (defined in Section B). Use a unique localStorage key per file (spark_theme_<context>).
